Admin App

Purpose: For the BringMe Delivery team

Overview: This application's software was made to track all the orders placed by app users; displaying where/when the order was made, when/where they were delivered, etc. It was supposed to be used to expand the company's delivery "bubble" and see where there can be improvement, but the company went in a different direction.

This project was very Javascript heavy, with multiple controllers being used. We also programmed the backend using C#, and even did the SQL database work - but I had less to do with the back-end.

This project uses Angular and Ionic frameworks, as well as package managers and organizers such as Bower, Yeoman, and Gulp.

Due to company protocol, I couldn't put the exact project online, but I've recreated the functionalities (without user data) and it can be found on my Github account under the "Admin-App" repository.

My Portfolio Website!

Purpose: To display my work

Overview: The is just a simple website intended to allow potential employers to understand what kind of work I've already done.

This website was made without a smart IDE - only sublime. I made it very simple, all using only HTML and CSS (except for the code to make this pop-up..). It's the first project I've ever done outside of a typical IDE.
I feel that a lot of times developers can get lost in all the new technologies (wireframes, libraries, etc) so my goal with this website was to go back to the roots & make it as "bare-bones" as possible.

The code can be found on my Github account under the "username.github.io" repository.

Image App

Purpose: To learn/for fun

Overview: It was essentially supposed to be "Instagram on a map". You can login and post photos that are attached to a location, or view all photos on a map without logging in.

Other than the basic HTML, and JavaScript on the page, I've used Google Maps APIs as well as a FireBase backend to deal with saving users and verifying logins.

This project also uses Angular and Ionic frameworks, as well as package managers and organizers such as Bower, Yeoman, and Gulp.

It can be found on my Github account under the "MyMap" repository.

Wordpress E-Commerce Site

Purpose: For a client to be able to sell their products.

Overview: A fellow classmate was helping a family member overseas create a site to allow them to sell their product. They sought out my help with general Wordpress knowledge, and then employeed me to help specifically with the front end of the project, resulting in my co-CTO & CDO of the company's tech.

The site was made using Wordpress. We ran the files on a local server and were altering the theme specifics. The client had a very detailed idea of what they wanted the website to look like, and how everything on the page should be organized.

Aside from my existing HTML/CSS/JavaScript skills, I had to learn how to work with plugins like image carousels and shopping carts. It was a great learning experience.

The website is live! It can be found here.
